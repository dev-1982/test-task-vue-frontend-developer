Учетные записи — техническая документация
Проект реализует интерфейс управления учетными записями с динамическим списком, валидацией и сохранением состояния. Архитектура построена вокруг предсказуемых данных, строгой типизации и изолированной логики компонентов. Визуальная структура повторяет предоставленный макет, при этом UI‑реализация выполнена на Element Plus.

Технологический стек
Vue 3 (Composition API) — основа компонентной модели

TypeScript — строгая типизация и контроль структуры данных

Pinia — централизованное состояние и персистентность

Element Plus — UI‑компоненты

Vite — dev‑сервер и сборка

Функциональность
1. Динамический список учетных записей
Пользователь может добавлять и удалять записи. Каждая запись — отдельная форма с собственным состоянием и валидацией.

2. Валидация полей
Валидация запускается:

blur — для текстовых полей

change — для селекта

Правила:

Поле	Условия
Метки	необязательное, ;‑разделитель, ≤ 50 символов на элемент
Тип записи	LDAP или Local
Логин	обязательное, ≤ 100 символов
Пароль	обязательное только для Local, ≤ 100 символов
Ошибочные поля подсвечиваются.

3. Парсинг меток
Строка "AAA; BBB; CCC" преобразуется в:

ts
[
  { text: "AAA" },
  { text: "BBB" },
  { text: "CCC" }
]
4. Логика пароля
LDAP → поле скрыто, значение сохраняется как null

Local → поле отображается и валидируется

5. Сохранение состояния
Все данные хранятся в Pinia и синхронизируются с localStorage.
При перезагрузке страницы список восстанавливается автоматически.

Архитектурные принципы
Компонентная изоляция  
AccountRow.vue полностью отвечает за состояние и валидацию одной записи.

Единый источник истины  
Все изменения проходят через Pinia, что обеспечивает предсказуемость.

Минимизация побочных эффектов  
Сохранение происходит только после успешной валидации.

Чистые утилиты  
Парсинг меток вынесен в отдельный модуль.

Строгая типизация  
Типы описаны в types/account.ts, что исключает расхождения между компонентами.

Запуск проекта

Установка зависимостей

npm install

Запуск dev‑сервера

npm run dev

Если требуется проброс хоста:

npm run dev -- --host

Сборка

npm run build

Почему эта реализация устойчива
Легко расширяется (добавление новых полей, интеграция API).

Валидация прозрачна и контролируема.

Состояние предсказуемо и не размазано по компонентам.

Кодовая база структурирована так, что новый разработчик быстро включится.

UI-слой отделён от бизнес-логики, что упрощает поддержку.
